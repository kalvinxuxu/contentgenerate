# 需求文档 (REQUIREMENTS.md)

## 1. 功能需求

### 1.1 核心功能

#### FR-1: 多 Agent 协作工作流
- **描述**: 系统应支持由多个 agent 按顺序协作完成文案生成
- **优先级**: P0
- **验收标准**:
  - [ ] 调研 agent 能够提取产品/主题的关键信息
  - [ ] 创作 agent 能基于调研报告生成文案初稿
  - [ ] 审核 agent 能识别文案中的问题并给出修改建议
  - [ ] 优化 agent 能根据审核意见改进文案

#### FR-2: 多文案类型支持
- **描述**: 系统应支持生成不同类型的文案
- **优先级**: P0
- **验收标准**:
  - [ ] 支持社交媒体文案（微信公众号、小红书）
  - [ ] 支持营销文案（产品详情页、电商海报）
  - [ ] 支持专业文章（新闻稿、官方博客）

#### FR-3: 风格配置
- **描述**: 用户可配置文案的风格和调性
- **优先级**: P1
- **验收标准**:
  - [ ] 支持选择正式/非正式语气
  - [ ] 支持选择文案长度（短/中/长）
  - [ ] 支持自定义品牌调性关键词

### 1.2 辅助功能

#### FR-4: 输入模板
- **描述**: 提供结构化的输入表单帮助用户提供信息
- **优先级**: P1
- **验收标准**:
  - [ ] 产品信息输入模板
  - [ ] 目标受众描述模板
  - [ ] 关键卖点输入框

#### FR-5: 输出管理
- **描述**: 管理和导出生成的文案
- **优先级**: P2
- **验收标准**:
  - [ ] 保存生成历史
  - [ ] 支持导出为文本文件
  - [ ] 支持对比不同版本

## 2. 非功能需求

### 2.1 性能需求
- **NFR-1**: 单次文案生成流程应在 2 分钟内完成
- **NFR-2**: 系统应支持并发处理多个生成请求

### 2.2 质量需求
- **NFR-3**: 生成的文案应语法正确、无明显错误
- **NFR-4**: 文案应符合指定的品牌调性

### 2.3 成本需求
- **NFR-5**: 单次生成的 API 成本应控制在合理范围内
- **NFR-6**: 应支持使用不同价位的模型

## 3. 技术需求

### 3.1 架构需求
- **TR-1**: 使用 LangChain 框架进行 agent 编排
- **TR-2**: 使用 Claude API 作为主要 LLM 提供商
- **TR-3**: 代码应符合 Python 最佳实践

### 3.2 可维护性需求
- **TR-4**: 每个 agent 应有清晰的职责边界
- **TR-5**: 应提供完整的日志记录
- **TR-6**: 关键配置应与代码分离

## 4. 用户故事

### 用户故事 1: 市场人员生成产品推广文案
> 作为一名市场人员，我希望输入产品信息后能快速生成多个版本的推广文案，以便在不同渠道进行测试。

**验收条件**:
- 输入产品名称、特点、目标受众
- 选择文案类型（如：小红书文案）
- 获得 3 个不同风格的文案版本

### 用户故事 2: 内容创作者生成博客文章
> 作为一名内容创作者，我希望基于一个主题生成完整的博客文章大纲和初稿，以提高写作效率。

**验收条件**:
- 输入文章主题和核心观点
- 选择文章类型（专业文章）
- 获得结构完整的文章初稿

### 用户故事 3: 品牌管理员确保文案调性
> 作为一名品牌管理员，我希望确保所有生成的文案都符合品牌调性，以维护品牌形象的一致性。

**验收条件**:
- 配置品牌关键词和禁用词
- 审核 agent 检查文案是否符合要求
- 输出合规性报告

## 5. 范围界定

### In Scope (范围内)
- 多 agent 工作流的核心实现
- 三种文案类型的支持
- Claude API 集成
- 基础配置管理

### Out of Scope (范围外)
- 用户界面（CLI 为主）
- 多模型路由（仅 Claude）
- 数据库存储（文件存储）
- 用户认证系统

## 6. 依赖项

1. **外部依赖**:
   - Python 3.10+
   - LangChain 库
   - Anthropic API (Claude)

2. **配置依赖**:
   - ANTHROPIC_API_KEY 环境变量
