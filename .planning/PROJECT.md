# 多 Agent 驱动的自动化文案生成工作流

## 项目概述

构建一个由多个 AI agent 协作的自动化工作流，用于生成不同类型的宣传文案。系统采用流程式分工，每个 agent 负责文案生成 pipeline 中的特定环节。

## 核心需求

### 目标文案类型
- **社交媒体文案**：微信公众号、小红书等平台
- **营销文案**：产品详情页、电商海报等
- **专业文章**：新闻稿、官方博客等

### 工作流程
1. **调研 Agent** - 分析产品/主题信息，收集关键点
2. **创作 Agent** - 基于调研结果生成初稿
3. **审核 Agent** - 检查文案质量、合规性、品牌调性
4. **优化 Agent** - 根据审核意见进行润色和改进
5. **配图 Agent** - 为最终文案生成 Midjourney 提示词和排版建议

### 技术要求
- 输出语言：中文
- 技术栈：Python + LangChain
- LLM：Claude API
- 图像生成：Midjourney API（用户自行调用）

## 架构设计

### Agent 角色定义

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 调研 Agent | 信息收集、关键点提取 | 产品/主题描述 | 调研报告 |
| 创作 Agent | 文案初稿撰写 | 调研报告 + 风格要求 | 文案初稿 |
| 审核 Agent | 质量检查、合规性审核 | 文案初稿 | 审核意见 |
| 优化 Agent | 润色改进、风格调整 | 初稿 + 审核意见 | 最终文案 |
| 配图 Agent | 视觉创意、MJ 提示词生成 | 最终文案 + 情绪基调 | 3 组 MJ 提示词 + 排版建议 |

## 成功标准

1. **功能性**：能够端到端生成完整的文案
2. **质量**：生成的文案符合品牌调性、语法正确、有吸引力
3. **效率**：单次生成流程在合理时间内完成（<2 分钟）
4. **可扩展性**：易于添加新的 agent 角色或文案类型

## 风险与挑战

- **上下文管理**：多 agent 传递信息时的上下文保持
- **风格一致性**：确保最终文案符合品牌调性
- **成本控制**：多次 LLM 调用的 API 成本优化
