## CLI 命令

### `generate` - 生成完整文案

运行完整工作流（调研→创作→审核→优化→配图）：

```bash
python main.py generate \
  --topic "AI 写作工具" \
  --audience "自媒体创作者" \
  --platform xiaohongshu \
  --tone casual \
  --emotion hopeful \
  --style minimalist \
  -k 关键词 1 -k 关键词 2 \
  -o output/result.json
```

**参数说明：**

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--topic`, `-t` | ✓ | - | 内容主题 |
| `--audience`, `-a` | | 职场人士 | 目标受众 |
| `--platform`, `-p` | | xiaohongshu | 发布平台：`xiaohongshu`, `wechat`, `blog` |
| `--tone` | | casual | 语气风格：`casual`, `formal`, `passionate` |
| `--priority` | | standard | 优化优先级：`quick`, `standard`, `deep` |
| `--emotion` | | hopeful | 情绪基调 |
| `--style` | | minimalist | 视觉风格偏好 |
| `--brand-keywords`, `-k` | | [] | 品牌关键词（可多个） |
| `--output`, `-o` | | - | 输出文件路径 |
| `--verbose`, `-v` | | false | 详细输出 |

---

### `research` - 仅运行调研 Agent

单独执行市场调研和用户分析：

```bash
python main.py research \
  --topic "时间管理" \
  --audience "职场人士" \
  --platform xiaohongshu
```

**输出内容：**
- 趋势分析
- 用户痛点
- 选题切入点（含置信度评分）

---

### `review` - 审核文案

对已有文案进行质量审核：

```bash
python main.py review draft.txt
```

**支持格式：**
- 纯文本文件
- JSON 格式（包含 `body` 或 `content` 字段）

**输出内容：**
- 总体评分（0-6 分）
- 维度评分（价值性、可读性、可信度等）
- 亮点列表
- 必须修改的问题

---

### `config` - 显示配置

查看当前各 Agent 的模型配置：

```bash
python main.py config
```

## 工作流说明

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  调研 Agent  │ →  │  创作 Agent  │ →  │  审核 Agent  │
│  Research    │    │  Creator     │    │  Reviewer    │
└──────────────┘    └──────────────┘    └──────────────┘
                                               │
                                               ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  输出结果    │ ←  │  配图 Agent  │ ←  │  优化 Agent  │
│              │    │  Image       │    │  Optimizer   │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 各 Agent 职责

| Agent | 职责 | 输出 |
|-------|------|------|
| **调研** | 市场分析、痛点挖掘、选题策划 | 趋势报告、用户画像 |
| **创作** | 基于调研生成文案初稿 | 完整文案 |
| **审核** | 质量评估、问题诊断 | 评分报告、修改建议 |
| **优化** | 根据审核意见润色改进 | 优化后文案 |
| **配图** | 生成 Midjourney 提示词 | 3 组配图方案 |

## 代码调用

在 Python 代码中直接使用：

```python
from src.utils.llm_client import create_llm_client, call_llm_sync

# 创建客户端
client = create_llm_client(
    model="qwen3.5-plus",
    temperature=0.7,
    max_tokens=2000
)

# 调用 API
response = call_llm_sync(
    client,
    system_prompt="你是一个专业的文案写手",
    user_prompt="请为 AI 写作工具写一篇小红书文案"
)

print(response)
```

## 配置文件

### 模型配置

各 Agent 的模型配置可在 `src/utils/config.py` 中自定义：

```python
AGENT_MODEL_CONFIGS = {
    "research": {"model": "qwen3.5-plus", "temperature": 0.7},
    "creator": {"model": "qwen3.5-plus", "temperature": 0.8},
    "reviewer": {"model": "qwen3.5-plus", "temperature": 0.5},
    "optimizer": {"model": "qwen3.5-plus", "temperature": 0.7},
    "image": {"model": "qwen3.5-plus", "temperature": 0.8}
}
```

## 常见问题

### 1. API Key 无效

检查 `.env` 文件中 `DASHSCOPE_API_KEY` 是否正确设置。

### 2. 中文乱码

终端编码问题，建议使用 UTF-8 兼容的终端。

### 3. 输出文件未生成

确保输出路径的父目录存在，或使用绝对路径。
